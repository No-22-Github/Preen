# 转换器零依赖化报告 — 消除 fla-hub 参照模型下载依赖

> 日期：2026-07-12 ｜ 范围：`tools/convert_rwkv7_to_hf.py` CLI + 仓库内 fixture/tokenizer vendor
> 验收：全新参数形式 `python tools/convert_rwkv7_to_hf.py --rwkv7 X.pth --output DIR` 在本机跑通；旧版显式参数与新版缺省参数转换产物逐项等价；pytest 全绿。

---

## 背景

`tools/convert_rwkv7_to_hf.py` 原本要求用户提供两个来自 fla-hub 0.1B 仓库的输入：

- `--reference`：0.1B `model.safetensors`，仅用于键名存在性 + ndim 校验，**不参与转换计算**。
- `--tokenizer-src`：拷贝 5 个 tokenizer 文件到输出目录。

新用户为了一次 sanity check 和 5 个小文件，被迫多下载一个完整模型仓库（含 364M 权重）。
本次把校验模板冻结成仓库内 JSON fixture、tokenizer 文件 vendor 进仓库，
用户转换模型时只需提供 `--rwkv7` 和 `--output` 两个参数。

**执行顺序的硬约束**：fixture 必须在删除 fla-hub 目录**之前**用活模型对账完成（T1 自检 + T4 等价性验收），删除后无回退机会。

## 1. T1 — 校验 fixture

**改动**：
- `tools/convert_rwkv7_to_hf.py`：把 `load_reference_template` 内的键名归一化（`model.layers.N.X` → `model.layers.{N}.X`）拆成纯函数 `normalize_layer_key(k)`，供 fixture 生成脚本复用。`load_reference_template` 行为不变。
- 新增 `tools/gen_convert_fixture.py`：一次性脚本，读 fla-hub 0.1B `model.safetensors`，复用 `normalize_layer_key`，导出只含 ndim 的 JSON（不存 shape——0.1B 与目标模型 hidden_size 不同，绝对 shape 无法比对，忠实于现状）。
- 产物 `tools/fixtures/rwkv7_hf_template.json`：35 个层内键 + 4 个顶层键，1880 字节。

**不改**：
- 不"顺手加强"成 shape 校验。fixture 只存 ndim，与现有校验逻辑（只比 `len(ref_shape)`）口径一致。
- 校验判定逻辑一行不动（见 T3）。

**自检**：脚本生成 fixture 后立即对账——fixture 的键集合 + ndim 与活 `model.safetensors` 经 `load_reference_template` 等价路径产出完全一致。断言通过。

## 3. T3 — 转换器 CLI 改造

**改动**：
- `--reference` 改为可选：缺省时从 `tools/fixtures/rwkv7_hf_template.json` 加载 fixture 走同样校验路径；显式提供时行为与旧版完全一致（活模型优先，作为上游 schema 漂移时的逃生通道）。
- `--tokenizer-src` 改为可选：缺省指向 `assets/rwkv_world_tokenizer/`。
- 新增 `load_template_from_fixture(fixture_path)`：读 fixture JSON，把每个 ndim 包装成 `(0,) * ndim` 的伪 shape，使校验判定处的 `len(ref_shape) == weight.ndim` 对 fixture 和活模型**完全同构**。
- `convert()` 签名：`ref_path`/`tokenizer_src` 默认 `None`，缺省时走内置路径。
- docstring 用法示例更新为两参数最短形式，并保留一行说明两个可选参数的用途。

**不改（一行不动）**：
- 校验判定逻辑：ndim 比对、`pre_norm` 允许缺失、`uncovered` 警告。
- `translate`（键名映射规则）、`infer_config`、dtype 处理、`config.json` 生成、`model.safetensors` 保存。
- 不删除 `--reference` / `--tokenizer-src` 参数（保留为可选覆盖）。

**定位**：fixture 和 tokenizer 缺省路径用 `os.path.dirname(os.path.abspath(__file__))` 相对脚本所在位置解析，不依赖 cwd。

## 4. T4 — 等价性验收（核心验收）

同一份 `rwkv7-g1d-0.4b-20260210-ctx8192.pth`、同一 `--precision bf16`，两次转换：

- **旧版**：从 git HEAD 取回旧转换器，显式参数（活模型 reference + fla-hub tokenizer）→ `/tmp/conv_eq_old/`。
- **新版**：改造后缺省参数（内置 fixture + vendored tokenizer）→ `/tmp/conv_eq_new/`。

两次 stdout 的校验段完全一致：795 张量，35 层内键 + 4 顶层键，同样的 3 个 `v_lora` layer0 未覆盖警告（机制不变）。

**逐项比对**：

| 项 | 比对方式 | 结果 |
|---|---|---|
| `model.safetensors`（文件级） | sha256 | ✅ `dcc3353f...dcef5` 两版完全一致 |
| `model.safetensors`（张量级） | 逐张量 `torch.equal`（795 个） | ✅ 全部数值一致 |
| `config.json` | `diff` | ✅ 完全一致 |
| tokenizer 5 文件 | sha256（新版产物 vs fla-hub 源） | ✅ 5 个全一致（见 T2 表） |

任何不一致都是失败——本次全绿，无解释性放行。

**pytest 兜底**：`.venv/bin/python -m pytest -q` → 73 passed, 3 skipped（推理 golden 用的已转换模型未被本任务触碰，无影响）。

T4 等价性验收全绿后，删除整个 `models/fla-hub-rwkv7-0.1B-g1/`：
