## 前置检查 C

### C1 — 安装官方 rwkv pip 包　**[通过]**

- `uv pip install rwkv` → `rwkv==0.8.32`（Author: Bo PENG）
- 包路径：`.venv/lib/python3.11/site-packages/rwkv/`，核心 `model.py`(85KB)

### C2 — rwkv pip 加载 0.4b，cpu fp32 forward　**[通过]**

**关键事实：rwkv pip 0.8.32 对 x070 有两条加载路径，必须 `RWKV_V7_ON=1`：**

| 路径 | 类 | x070 表现 |
|---|---|---|
| 默认 | `class RWKV`(model.py:612) | **失败**：x070 权重无 `time_maa` 键(用 `x_r/x_w/...` 形式)，版本判 `version=5`，forward 崩 `args.n_head` 缺失 |
| `RWKV_V7_ON=1` | `class RWKV_x070`(model.py:223) | **通过**：经 `blocks.0.att.r_k` 识别 |

C2 通过证据：n_layer=24, n_head=16, head_size=64，forward logits shape `[65536]`，state len=72=24×3，att_kv shape `(16,64,64)`。

> ⚠️ 此事实一度误导诊断：rwkv pip 0.8.32 把 x070 判为 version 6.0（无 v7 赋值），而 **RWKV-Runner
> 自带的 model.py 把 x070 判为 version 7**（有 `self.version=7` 赋值，见任务2）。这两个版本对
> state 的 transpose 处理不同，是整个方向错误的根源。

### C3 — 官方 state 加载代码原文 + v7 结构核对　**[通过]**

rwkv pip 0.8.32 model.py:1460-1471（主 class RWKV，v5/v6 分支）含 `.time_state` 加载 +
`.transpose(1,2)`。但 x070 走 `RWKV_x070` 类，其 `generate_zero_state`(line 284-290) **不读
`.time_state`**，只生成全零。即 rwkv pip 0.8.32 无现成的「加载 x070 + 读自定义 state」路径。

**✅ 最终补充**：真正的官方加载逻辑在 RWKV-Runner（见任务2），不在 rwkv pip 包。

### C4 — 官方 transpose 原文（rwkv pip 侧）　**[作废，见任务2修正]**

rwkv pip model.py:1468 的 `.transpose(1,2)` 是 **v5/v6 路径**。初稿误将其当作所有版本的通用
逻辑，得出"导出器应 swapaxes"的错误结论。**x070(v7) 在 RWKV-Runner 中不 transpose**（任务2铁证）。

## 实验 A：MLX(kernel) vs rwkv pip(RWKV_x070) 末态 state 对比

脚本：`experiments/p0_translate/exp_a_direction.py`　结果：`exp_a_direction_result.json`
3 句取自 test_10.jsonl 前三条（训练集外，tokenizer 全对齐）。

**结果：direct(原样) mean 3.09e-4 vs transposed mean 2.90e-2，比值 93.78× → 原样匹配。**

24层×3句全量误差表见 `exp_a_direction_result.json`。

### ⚠️ 实验 A 的有效性边界（修正后明确）

实验 A 证明的是：**MLX kernel 与 rwkv pip RWKV_x070 两个引擎 forward 后的"末态 state"同向（原样匹配）**。
这个结论本身正确且可信——CUDA kernel 源码(rwkv7.cu:46-52)也独立印证了三方同向。

**但实验 A 无法回答的问题**：我们训练的初始 state 经导出→Runner 加载后，落在哪个方向？
实验 A 测的是 forward 产出的末态，不是导出链路。初稿错误地用"末态同向"推论"导出方向正确"，
跳过了导出器↔Runner 这关键一环。**Windows 真机验收正是补上了这一环，推翻了推论。**

## 任务2（修正追加）：RWKV-Runner x070 state 加载源码　**[决定性铁证]**

来源：`RWKV-Runner/backend-python/utils/rwkv.py` 的 `load_rwkv_state`（line 836-857）：

```python
for i in range(args.n_layer):
    ...
    if model.model.version >= 7:                          # ← x070 (line 843)
        model.state_tuned[i * 3 + 1] = (
            state_raw[f"blocks.{i}.att.time_state"]
            .to(dtype=torch.float, device=dev)            # ← 原样，不 transpose!
            .requires_grad_(False).contiguous()
        )
    else:                                                  # ← v5/v6 (line 850)
        model.state_tuned[i * 3 + 1] = (
            state_raw[f"blocks.{i}.att.time_state"]
            .transpose(1, 2)                              # ← 只 v5/v6 转!
            .to(dtype=torch.float, device=dev)
            ...
        )
```

Runner 自带 model.py(line 1091) 把 x070 判为 **version=7**（`self.version = 7`）。

**✅ 最终结论：x070 模型在 RWKV-Runner 中加载 state 时 NOT transpose。导出器必须存原样，
不做 swapaxes。初版 export.py 的 swapaxes 是按 v5/v6 设计的错误假设。**

## 修正记录

**2026-07-10 结论反转：**

| 项 | 初稿结论（作废） | 最终结论 |
|---|---|---|
| 导出方向 | swapaxes 正确（v5/v6 假设） | **noswap 正确（x070 Runner 不 transpose）** |
| 证据 | 实验 A direct 匹配 + rwkv pip model.py:1468 | **RWKV-Runner rwkv.py:843 version>=7 分支 + Windows 真机** |
| export.py | 无需改 | **已改：x070=True 默认不 swapaxes** |
| "精确 0.0" round-trip | 作为正确性证据 | **同义反复（转两次必还原），不证明方向正确，标记为不可信** |

**根因**：初稿把 rwkv pip 0.8.32 的 v5/v6 transpose 逻辑误当作 x070 通用逻辑，未查 RWKV-Runner
源码。Windows 真机验收（官方 state 能过、我们的碎渣）是推翻错误结论的关键外部证据。

**教训**：Mac 侧闭环验证（round-trip / 挂载等价性）只能证明"内部自洽"，不能证明"与外部消费者
兼容"。涉及导出格式的结论，必须以外 部真实环境验收为准。
