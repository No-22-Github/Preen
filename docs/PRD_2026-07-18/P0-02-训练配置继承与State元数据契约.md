# P0-02：训练配置继承与 State 元数据契约

> 优先级：P0  
> 状态：已评审，待实施  
> 所属版本：v1.1.0

## 一、问题

从训练完成页或训练记录进入对话时，当前只传递训练模型路径和 State 路径。训练使用的 `template` 没有进入对话配置，可能出现“按 instruction 训练、按 qa 验证”的错误口径。

外部登记的 State 还可能缺少完整 run 记录，因此验证配置不能只依赖历史数据库，还需要稳定的 State metadata 契约。

## 二、目标

- 训练与推理自动使用同一模板。
- 从训练记录、State metadata 中恢复初始建议，同时保留用户最终调整权。
- 老版本 State 仍然可用，但缺失字段必须对用户可见。
- State 与模型结构不兼容时在加载前阻断；模型名不同时给出信息提示。

## 三、配置来源优先级

加载 State 时，先按以下顺序得到初始建议：

1. 当前训练记录的不可变 config 快照。
2. State 同目录的 `.meta.json`。
3. App 默认配置。

高优先级来源存在时，界面展示“建议来自训练记录”或“建议来自 State 元数据”；不允许静默被 App 默认值覆盖。用户随后明确修改并确认的配置始终最高优先，不再被记录或 metadata 改回。

## 四、元数据字段

在既有 metadata 基础上保证以下字段可读取：

```json
{
  "format_version": 2,
  "model_name": "rwkv7-g1d-0.4b",
  "model_path": "...",
  "template": "qa",
  "data_sha256": "...",
  "state_format": "npz",
  "state_dtype": "float32",
  "created_at": 1784300000.0
}
```

说明：

- reasoning / think 属于推理方言，不写入 State 训练 metadata；已保存的 A/B 结果单独记录当次实际使用值。
- RWKV 原生 `.pth` 没有统一、可信的模型指纹契约。v1.1 只记录模型目录的展示名 `model_name`；`model_path` 仅供本机找回原模型，不作为身份证明。
- 模型名相同不代表权重一定相同；界面不把名称匹配包装成安全验证。
- State 与模型是否可加载，仍由后端对层数、层号和每层 shape 做结构校验。
- v2 新写入 `model_name / model_path / state_format / state_dtype`；读取旧 v1 时把现有 `model` 字段视为历史 `model_path`，并从路径末段派生展示名。
- 新字段必须保持向后兼容，Swift 对旧 metadata 的非必需字段可选解码；不因旧文件缺少新字段而整体解码失败。

## 五、交互

从完成页或历史记录点击“比较效果 / 去对话”时：

1. 恢复训练模型。
2. 恢复训练模板。
3. 加载 State。
4. 保留用户当前的 reasoning / think；若尚无用户设置，使用 P0-01 的安全默认。
5. 在对话页显示配置来源。

如果 State 缺 metadata：

- 显示“未找到模板信息”。
- 让用户选择 QA / Instruction / Raw。
- 用户确认前不自动开始 A/B 生成。

如果当前模型名与 metadata 中的 `model_name` 不同：

- 显示当前模型与训练模型摘要。
- 提供“切换到训练模型”和“仍用当前模型”两个动作。
- “仍用当前模型”需要明确说明“名称不同，只能在加载时继续验证结构”，但不禁止高级用户继续。

无论模型名是否相同，结构校验失败时都必须阻断加载，不提供强行继续。

## 六、验收标准

- [ ] `instruction` 训练记录点击“去对话”后，serve session 的 template 为 `instruction`。
- [ ] QA 训练记录继续使用 `qa`，既有 NekoQA 路径不回退。
- [ ] 外部 State 有 metadata 时能恢复模板。
- [ ] 外部 State 无 metadata 时要求用户确认模板，不静默假设。
- [ ] 老 metadata 解码不崩溃，缺失字段有清晰降级文案。
- [ ] 用户手动修改配置并确认后，训练记录或 metadata 不会再自动改写该选择。
- [ ] 模型名不同时在生成前给出信息提示；结构不兼容时阻断加载。
- [ ] 切换模型、模板或 State 后，顶部会话摘要与后端实际配置一致。

## 七、不做

- 不通过 State 数值内容猜测其训练模板。
- 不把模型名或本机绝对路径宣称为可靠指纹。
- 不在 v1.1 对数 GB 权重计算全量哈希。
- 不自动迁移或覆盖用户已有 metadata 文件。
