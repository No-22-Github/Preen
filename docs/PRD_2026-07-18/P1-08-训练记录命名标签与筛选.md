# P1-08：训练记录命名、标签与筛选

> 优先级：P1  
> 状态：已评审，待实施  
> 所属版本：v1.1.0

## 一、问题

训练记录列表主要显示数据文件名、时间、状态和最终 loss。重复实验经常使用同一个数据文件，列表会出现大量相同名称，用户难以区分不同模型、ctx、seed 或实验意图。

## 二、目标

- 让用户能快速识别每次实验。
- 不破坏 run ID 与文件目录的稳定身份。
- 支持按常用维度筛选，而不是把历史页做成复杂数据库。

## 三、命名

### 3.1 自动显示名

默认名称组合：

```text
<数据名> · <模型名> · <模板>
```

列表副行继续显示时间、状态和关键结果。

### 3.2 用户自定义名称

- 记录详情支持“重命名…”。
- 自定义名称只写入 run metadata，不重命名原数据、State 或 run 目录。
- 保留原始数据名作为详情字段。

### 3.3 标签

支持零到多个本地标签，例如：

- baseline
- lr-test
- nekoqa
- release-candidate

首版仅支持文本标签，不支持颜色和层级。

## 四、筛选与排序

筛选：

- 状态。
- 模型。
- 模板。
- 标签。

排序：

- 创建时间，默认最新优先。

v1.1 不提供跨记录的最终 loss / 最佳验证 loss 排序。loss 只在记录自身的详情中展示，避免把不同数据、模型或模板的数值暗示为可比排名。后续若提供实验组功能，再限定为同数据 SHA、模型名与模板内的对比。

搜索覆盖自定义名称、数据名、模型名、State 文件名和标签。

## 五、列表信息

列表主行：自定义名称或自动显示名。  
副行：时间 · 状态 · 模型档位。  
尾部：最终 loss 或失败摘要。

不在列表直接显示 UUID；UUID 保留在详情和诊断中。

## 六、验收标准

- [ ] 同一数据文件的不同模型实验可以在列表中区分。
- [ ] 重命名后重启 App 仍保留。
- [ ] 重命名不改变 run ID、目录或产物路径。
- [ ] 标签可添加、删除、搜索和筛选。
- [ ] 外部登记 State 也能使用名称和标签。
- [ ] 筛选无结果时显示明确空态和“清除筛选”。
- [ ] 列表不提供跨数据/模型的 loss 排序入口。

## 七、不做

- 不做云同步或跨设备协作。
- 不做标签颜色、嵌套文件夹和智能集合。
- 不允许修改训练事实字段。
