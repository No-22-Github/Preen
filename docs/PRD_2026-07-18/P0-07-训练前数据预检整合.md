# P0-07：训练前数据预检整合

> 优先级：P0  
> 状态：已评审，待实施  
> 所属版本：v1.1.0

## 一、问题

训练配置页当前只读取前几条原始记录，明确不包含模板渲染、token 长度与截断风险。工具箱的数据集预览已经能提供这些信息，但要求用户离开训练主线进入另一个工具。

两套预览口径并存，会让用户误以为配置页看到的文本就是模型实际接收的文本。

## 二、目标

- 将工具箱预检能力整合到训练配置页。
- 展示最终模板文本，而不是原始 JSON 字段。
- 在开始训练前明确 schema、token 分布和截断风险。
- 工具箱继续保留完整分页预览，但不再是训练前检查的唯一入口。

## 三、配置页信息层级

### 3.1 默认可见摘要

- 探测格式与置信度，例如 `Alpaca · 100%`。
- 有效样本数 / 总样本数。
- 平均 token、P95、最大 token。
- 仅截去部分前缀的样本数、target 完全截断数。两者在 UI 中是互斥口径；若后端 `truncated` 包含后者，前者显示值为 `truncated - target_fully_truncated`。
- 训练 / 验证条数与预计步数。

### 3.2 展开预览

默认展示 3 条最终 RenderedSample：

- 输入前缀。
- 训练目标。
- stop token 是否追加。
- 当前总 token 数。
- 是否截断、截断发生在哪一段。

prefix 与 target 使用结构标签和分段背景，不只靠颜色。

### 3.3 未知格式

探测为 unknown 时，训练页不复制一套字段映射器。展示“在数据导入器中配置”操作，带上当前数据路径跳转工具箱现有导入流程；用户完成映射和导入后，自动回到训练页并选中新的标准产物。

## 四、运行策略

- 自动检查阈值在实施前用 macOS 14 / 26 的冷启动与大数据实测锁定，PRD 不预先把 30K 当作无实验依据的事实。
- 超过阈值时关闭因 ctx/模板变动触发的即时自动重算，显示数据规模与“运行完整检查”按钮。用户仍需完成一次与最终配置同口径的完整检查后才可开始训练，不用抽样数据伪装全量统计。
- 改变 ctx、模板或多轮策略时使旧结果失效并重新检查。
- 复用工具箱预览缓存，避免相同模型 / 数据 / ctx / 模板重复 tokenize。
- 缓存键必须包含数据修改时间或 SHA、模型 tokenizer 身份、ctx、模板和映射配置。

## 五、截断交互

- 部分截断：黄色信息提示，说明“截头保尾”。
- target 完全截断：橙色强提示，建议增加 ctx 或丢弃超长样本。
- `drop_truncated=true`：显示实际丢弃数和重算后的训练步数。
- 截断仍按现有产品裁决只警告不阻断，除非有效训练样本归零。

## 六、验收标准

- [ ] 配置页展示的 prefix / target 与 Python `encode_template_sample` 同源。
- [ ] 同一条样本的分界与 `prefix_len` 一致。
- [ ] 改变 ctx 后 token 统计与截断数自动刷新。
- [ ] unknown schema 可带当前数据跳转工具箱完成映射，导入成功后返回训练页继续。
- [ ] 训练页和工具箱对同一输入给出一致的 schema 与统计。
- [ ] target 完全截断和部分截断有不同严重度。
- [ ] 大数据关闭即时自动重算后，未完成最终配置的完整检查前不能启动训练。
- [ ] 检查失败不伪装成成功；用户可以查看错误并重试。

## 七、不做

- 不实现数据集内容编辑。
- 不自动删除“质量差”的样本。
- 不根据文本语义给数据质量打分。
