# P0-08：训练结果可解释摘要

> 优先级：P0  
> 状态：已评审，待实施  
> 所属版本：v1.1.0

## 一、问题

训练完成页与记录详情已经有完整曲线和指标，但用户仍需要自己解释：loss 是否下降、最佳轮次在哪里、验证集是否改善、是否早停、数据是否发生截断。

产品不能替用户武断判定 State “好或坏”，但可以把客观变化整理成可读结论。

## 二、目标

- 在完成页 3 秒内读懂训练发生了什么。
- 只使用已定义指标，不新增未经标定的健康阈值。
- 将摘要与 A/B 验证连接起来。

## 三、摘要内容

### 3.1 训练结果

- 实际轮数 / 配置上限。
- 第 1 轮训练 loss、最终训练 loss、两者相对变化。第 1 轮是该轮完成后的平均值，不标成“训练前基线”。
- 最佳 held-out loss 与对应轮次。
- 结束原因：正常完成 / 早停 / 取消 / 失败。
- State 标准差：只记录，不评价健康或异常。
- 总耗时。运行中可继续显示近期 s/step 与 ETA，但训练结果不用包含 held-out 评估与收尾时间的总耗时倒算“平均 s/step”。

### 3.2 数据与配置

- 训练 / 验证样本数。
- 模板。
- ctx。
- 截断 / 丢弃数量。
- 模型与精度。
- `data_sha256` 短摘要。

### 3.3 下一步

- 主按钮：比较效果。
- 次按钮：在 Finder 中显示。
- 菜单：导出 `.pth`、查看完整曲线、复制诊断。

## 四、表述规则

- 允许：“验证 loss 从 1.46 降至 1.42”。
- 允许：“最佳验证 loss 出现在第 2 轮”。
- 允许：“因连续 3 轮未改善而早停”。
- 禁止：“训练成功率 92%”。
- 禁止：“State 非常健康”。
- 禁止使用未标定的 State std 红黄绿阈值。

若无 held-out 数据，只展示训练 loss 变化，并明确“未启用验证集，无法判断泛化趋势”。

若训练在第 1 轮完成前取消或失败，不显示 loss 变化、State std 或其他不存在的结果；只展示已发生的阶段、终止原因、已用时间与可用诊断。

## 五、验收标准

- [ ] 正常完成、早停、取消、失败四种状态都有对应摘要。
- [ ] “第 1 轮→最终轮”相对变化的计算口径在 Swift 测试中锁定，不称为训练前基线。
- [ ] 没有 held-out 数据时不显示虚构的验证结论。
- [ ] State std 只作为事实数值展示，不着色、不报警。
- [ ] 所有摘要数值可追溯到事件或 metadata。
- [ ] 点击“比较效果”使用本次训练的模型、模板和 State。

## 六、不做

- 不建立综合评分。
- 不自动选择最佳 checkpoint 替换最终 State，除非另立实验与产品裁决。
- 不对不同数据集之间的 loss 做横向排名。
