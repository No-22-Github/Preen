# P0-03：State 效果 A/B 对比

> 优先级：P0  
> 状态：已评审，待实施  
> 所属版本：v1.1.0  
> 依赖：P0-01、P0-02

## 一、问题

当前用户验证 State 的方式是先在基线模式聊天，再手动加载 State、重置会话并重新输入同一问题。这一过程容易出现 prompt、seed、采样参数或历史上下文不一致，无法形成可信对照。

State tuning 的产品价值是行为、风格与格式变化。A/B 对比应是默认验证方式，而不是隐藏在手工操作里的高级玩法。

## 二、目标

- 同一 prompt、同一 seed、同一生成参数下同时展示基线与 State 结果。
- 从训练完成页一次点击进入比较。
- 用户无需复制问题或切换 State。
- 保留技术摘要，帮助识别停止原因和性能差异。

## 三、入口

- 训练完成页：主按钮“比较效果”。
- 训练记录详情：将“去对话”从操作菜单提升为可见主操作“比较效果”。
- 对话页：提供 A/B 开关，默认在从训练链路进入时开启。
- State 工具栏：加载 State 后提供“开始比较”。

## 四、界面

双栏结构：

```text
无 State（基线）        有 State · state.npz
模型回答                模型回答
stop / tokens / t/s     stop / tokens / t/s
```

要求：

- 两栏宽度相等，窗口不足时允许垂直堆叠，不横向压缩到不可读。
- 右栏使用一条轻量 accent 边界表示 State，不靠大面积颜色。
- 两栏共享一个输入框和一个停止按钮。
- 两边都完成前允许分别显示进度与错误，不让一侧失败抹掉另一侧结果。
- 明确显示统一的模板、reasoning、think、seed 和采样参数。

## 五、测试 prompt

从训练完成或历史记录进入时，提供以下来源：

1. 用户手动输入。
2. 从训练数据原文件中临时读取 prompt，默认展示 3 个建议。
3. 最近使用过的比较 prompt。

首版不自动给出“哪边更好”的评分，避免将风格偏好伪装成客观指标。

训练数据建议不在训练时复制进 run 记录。原数据文件不存在或无法读取时，隐藏该组建议并保留手动输入，不因此阻断 A/B。

## 六、生成语义

- 同一轮两栏使用相同 seed 和 `gen_config`。
- 两侧不并行执行：先生成基线侧，再生成 State 侧；双栏从开始就同时展示，每侧出来多少就流式显示多少。
- 每侧开始前都使用同一 seed 重置采样器，不共享另一侧的随机状态或 cache。
- 基线侧 `state_path=nil`，State 侧使用选中 State。
- 两边使用同一 template / reasoning / think。
- 首版限定单轮独立比较，不继承普通聊天历史。
- 用户停止时，中断当前正在生成的一侧，保留已完成侧和当前侧的已生成文本，尚未开始的一侧标记为“已跳过”。

协议在现有 `preview(ab=true)` 上扩展 side-tagged 流式事件，一个请求内顺序跑完两侧。不在 Swift 侧建立两个同时生成的会话，不修改推理引擎为并行解码。每侧的 `text_chunk / turn_end / error` 均带 `side=baseline|with_state`，一侧失败后仍尝试另一侧。

## 七、结果保存

允许用户将一次比较结果保存到对应训练记录：

```text
prompt
baseline_text
state_text
template / reasoning / think
gen_config / seed
stop_reason / token_count / tps
created_at
```

默认不自动保存自然语言内容，用户主动点“保存比较”后才写入本地 run 目录。

## 八、验收标准

- [ ] 完成页到首次 A/B 生成只需一次点击和一次发送。
- [ ] 两栏收到的渲染 prompt 除 State 注入外逐 token 一致。
- [ ] 两栏 seed、temperature、top_p、惩罚参数完全一致。
- [ ] A/B 协议读取并回传 `max_tokens / temperature / top_p / seed / presence_penalty / frequency_penalty / penalty_decay` 七个字段，不把用户修改过的重复惩罚静默替换为后端默认值。
- [ ] 在同一设备、模型文件、App/后端版本与生成配置下，相同比较可复现；不承诺跨设备或跨版本字节级一致。
- [ ] 任一侧失败时另一侧结果仍可阅读，并显示独立错误。
- [ ] abort 后当前侧在下一 MLX step 边界停止，已完成文本不丢失，未开始侧不再启动。
- [ ] 从 `instruction` run 进入时两栏均使用 `instruction`。
- [ ] 没有 State 时 A/B 入口禁用并说明原因。
- [ ] 用户主动保存后，关闭并重开 App 仍能查看该次比较。

## 九、不做

- 不在 v1.1 做多轮双 cache A/B。
- 不用 LLM 自动判断哪一侧更好。
- 不把 loss 或文本相似度包装成“人格注入成功率”。
