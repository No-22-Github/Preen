# P0-05：内置 NekoQA 200 示例数据与首次训练

> 优先级：P0  
> 状态：已评审，待实施  
> 所属版本：v1.1.0  
> 需求类型：首次使用 / 示例内容 / 发行资源

## 一、问题

Preen 当前已经具备数据导入与格式探测能力，但新用户第一次训练前仍需自行寻找数据集、确认格式并选择文件。对于只想验证“State tuning 是否能工作”的用户，这部分准备成本与模型准备叠加，容易在看到产品核心价值前流失。

仓库现有 `train_data/NekoQA_10k/nekoqa_smoke_200.json`：

- 共 200 条 QA，体积约 66 KB；
- 是 NekoQA-10K 前 200 条的未修改子集；
- 已用于项目冒烟训练，和当前 QA 模板、测试基线一致；
- 原数据集采用 Apache License 2.0，仓库已有 LICENSE 与 NOTICE。

因此 v1.1 考虑把经审核的 NekoQA 200 条子集随 App 内嵌，让用户准备好可训练模型后，无需再准备数据即可完成第一次训练。

## 二、目标

- 用户准备好一个可训练的 BF16 模型后，可直接选择内置示例开始训练。
- 首次训练不再要求下载数据、理解数据格式或打开数据文件选择器。
- 示例清楚演示 State tuning 的能力边界：学习角色、语气与表达风格，不承诺学习新事实。
- 内置数据可追溯、可复现、带完整许可证归属，App 升级时不会静默变化。
- 不影响高级用户继续导入自己的数据。
- 用快速的真实训练链路展示产品价值，不以内嵌预训练 State 代替首次训练。

## 三、核心用户路径

### 3.1 欢迎页入口

当用户尚无训练记录时，欢迎页展示主路径：

1. 选择或准备 BF16 模型。
2. 点击「使用内置示例训练」。
3. 进入训练配置页，数据源已选为「内置示例 · NekoQA 200」。
4. 确认配置并开始训练。
5. 训练完成后进入 State 效果 A/B 对比。

在模型已经就绪的前提下，从欢迎页到训练配置就绪不超过 2 次点击，不弹出数据文件选择器。

### 3.2 训练页入口

训练配置页的数据选择区域同时提供：

- 「使用内置示例」：首次使用时作为推荐操作；
- 「选择自己的数据」：保留现有文件导入路径。

选中内置示例后显示只读信息：

- 名称：NekoQA 200；
- 样本数：200；
- 类型：角色风格 QA；
- 模板：QA；
- 来源与许可证入口；
- 一句话能力说明：“用于体验角色与表达风格迁移，不用于学习新知识。”

用户可以随时切换为自己的数据，切换行为遵循现有未保存配置确认规则。

### 3.3 模型不满足训练要求

若当前选择的是 INT8 模型或没有可训练模型：

- 入口仍可见，但点击后先引导用户选择 BF16 模型；
- 不把 INT8 模型带入训练页后再报错；
- 文案明确“量化模型可用于推理，训练需要 BF16 模型”。

本需求不包含模型在线下载；用户首次训练唯一仍需自行完成的准备是可训练模型。

## 四、内置数据版本方案

### 4.1 v1.1 首版候选

现有 `nekoqa_smoke_200.json` 只作为审查起点和回归基线，不因为其已用于冒烟训练就直接成为发行内容。发行子集必须完成下节审查后另存为带版本的固定文件。

当前候选基线：

| 项 | 值 |
|---|---|
| 样本数 | 200 |
| 文件大小 | 约 66 KB |
| 当前筛选规则 | 原始 NekoQA-10K 前 200 条，内容未修改 |
| SHA-256 | `c86f6ff4df41c4ebaa55952c72ce8de1f717801877c8ce8debb9ce1eda13b244` |
| 许可证 | Apache License 2.0 |

以上 SHA 只是当前冒烟基线的记录，不是 v1.1 发行文件的预先承诺。完成发布前内容审查后，以发行 manifest 中的值为最终事实源。

### 4.2 发布前内容审查

内嵌内容代表产品默认示例，不能仅因测试可运行就直接发布。发布前必须完成一次人工与自动审查：

- 结构合法：每条都有非空 `instruction` 与 `output`；
- 去重：无完全重复或近似重复到明显影响体验的样本；
- 安全与信任：排除个人隐私、露骨性内容、仇恨、危险行为指导以及医疗/法律/财务建议；
- 产品定位：保留能表达猫娘角色的称谓与擬人化互动，排除明显的恋人/婚姻关系、绝对服从、永不离开或鼓励情感依赖的条目；
- 展示质量：排查乱码、异常长文本、明显语病和脱离猫娘角色的输出；
- 训练有效性：使用 QA 模板与默认 ctx 预检，确认没有 target 完全截断；
- 覆盖度：至少覆盖问候、闲聊、情绪回应、行为描写与角色称谓等典型风格场景。

若现有前 200 条未通过审查：

- 从原始数据集中重新确定 200 条固定子集；
- 只筛选、不改写原条目内容；
- 用源数据索引清单记录选择结果，不用随机抽样结果作为隐式事实；
- 新子集使用独立文件名与版本号，避免静默替换测试基线；
- 是否同步更新现有 NekoQA golden 由单独变更评审决定。

### 4.3 发行资源结构

App 包内建议使用以下只读资源结构：

```text
Resources/Datasets/NekoQA200/
├── nekoqa_200.json
├── manifest.json
├── LICENSE
└── NOTICE.md
```

`manifest.json` 至少记录：

- `id` 与展示名称；
- 子集版本；
- 样本数；
- 文件 SHA-256；
- 原数据集名称、作者与来源 URL；
- 许可证标识；
- 筛选规则或源索引清单版本；
- 推荐模板；
- 内容审查日期。

在锁定 manifest 前，必须以上游当前数据集卡与引用信息复核作者、版权归属与 NOTICE；不直接复用现有 NOTICE 中可能过时的作者/年份字段。

运行记录中保存 `dataset_source=builtin:nekoqa_200`、子集版本和 SHA。训练进程可以读取 App bundle 内的只读文件，不在首次使用时复制出一个可被意外修改的隐式副本。

## 五、默认配置

选择内置示例时只自动填充与数据语义直接相关的配置，不建立另一套训练参数：

| 配置 | 默认行为 |
|---|---|
| 数据 | 内置 NekoQA 200 |
| 模板 | `qa`，自动选择并明确展示 |
| ctx | 512 |
| reasoning | 关闭；数据本身不绑定特定 reasoning 模型 |
| 学习率等训练参数 | 继续使用产品现有默认值，不为示例特调 |
| 输出位置 | 使用 P0-06 的自动生成规则 |
| 数据预检 | 使用 P0-07 的同源预检并展示摘要 |

如果用户修改模板或 ctx，内置数据仍按普通数据源处理，重新执行预检，不隐藏截断风险。

## 六、许可与归属

- App 包必须同时带上 Apache-2.0 LICENSE 与本子集 NOTICE。
- 数据详情页或“关于”中的第三方内容入口可查看名称、作者、来源和许可证。
- 不把 NekoQA 或原作者表述为 Preen 官方背书方。
- 发布包、开发包和两个系统版本构建产物中的数据文件必须字节一致。
- 许可证与 NOTICE 缺失时，发行构建检查失败。

## 七、衡量与隐私边界

核心结果指标：已有 BF16 模型的新用户，无需准备数据即可启动并完成第一次训练。通过发布前可用性测试与用户主动反馈衡量；v1.1 不引入联网埋点、遥测 SDK、上传队列或隐性事件采集。

## 八、验收标准

- [ ] 全新安装且已有可训练 BF16 模型时，用户不选择数据文件即可进入训练配置并启动训练。
- [ ] 从欢迎页点击入口到配置就绪不超过 2 次点击，期间不出现数据文件选择器。
- [ ] 内置数据预检结果为恰好 200 条有效样本，且默认 ctx 下 target 完全截断数为 0。
- [ ] 训练实际使用 `qa` 模板，运行记录写入内置数据 ID、版本和 SHA。
- [ ] 训练完成后可按 P0-03 直接进入同口径 State A/B 对比。
- [ ] 无网络环境下，只要模型已准备好，内置示例训练仍可开始。
- [ ] INT8 或未选择模型时给出 BF16 模型引导，不以通用训练失败结束。
- [ ] App 包内包含数据文件、manifest、LICENSE 与 NOTICE，发行检查能验证数量和 SHA。
- [ ] macOS 14 与 macOS 26 构建中的内置数据字节一致。
- [ ] UI 明确说明示例用途、数据来源、许可证以及“学风格、不学事实”的能力边界。
- [ ] 用户选择自己的数据时，现有导入、预检和训练流程不受影响。

## 九、发布门槛与回滚

### 9.1 发布门槛

- 数据内容审查有可追溯记录；
- 法务归属文件齐全；
- 发行构建能够校验固定 SHA；
- 至少完成一次从全新用户状态到训练完成再到 A/B 对比的端到端测试。

### 9.2 回滚原则

若发布前发现许可或内容风险，允许隐藏内置示例入口并回退为用户自选数据，不允许在没有版本记录的情况下替换包内条目。已发布后需要更换子集时，必须提升子集版本并在发行说明中披露。

## 十、不做

- 不内嵌完整 NekoQA-10K。
- 不在 App 内提供数据集编辑与标注功能。
- 不自动联网更新示例数据。
- 不在 v1.1 内嵌预训练 NekoQA State 代替真实训练。
- 不把示例训练结果宣传为通用模型能力 benchmark。
- 不更改 Trainer、loss、学习率或提前停止的现有产品默认口径。
- 不在本需求中实现模型下载或模型市场。
