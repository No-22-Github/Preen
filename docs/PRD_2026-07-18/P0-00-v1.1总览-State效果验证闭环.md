# Preen v1.1 PRD 总览：State 效果验证闭环

> 日期：2026-07-18  
> 版本建议：v1.1.0  
> 优先级：P0（版本总览）  
> 版本主题：训练完成后，一键比较基线与 State，让效果可见、口径可信  
> 状态：已评审，待实施

> 评审裁决（2026-07-18）：P0 扩为八项；内嵌经重新筛选的 NekoQA 200 而不内嵌预训练 State；不加埋点；reasoning / think 由用户明确选择；metadata 只提供初始建议且用户修改优先；模型只记录名称并继续做 State 结构校验；A/B 为顺序、逐侧流式生成。本组 PRD 在这些 v1.1 事项上优先于早期 Roadmap / Phase 3 草案。

## 一、背景

Preen v1.0 已经具备完整的模型转换、数据导入、State 训练、训练监控、历史记录、推理对话与 `.pth` 导出能力。技术底座和可观测性已经达到可正式使用的水平。

当前主要问题不再是“能不能训练”，而是“用户能不能低阻力开始，并快速、正确地判断训练是否有效”。现有首次使用与验证链路存在以下断点：

1. 训练记录进入对话时只传递模型与 State，没有完整继承训练模板。
2. App 没有暴露 reasoning / think，无法保证 reasoning 模型按官方格式推理。
3. 对话仍是单栏，用户需要手动加载、卸下 State 才能对比。
4. 训练配置页展示原始字段，但不展示最终模板文本与 token 风险。
5. 训练结果主要呈现曲线，没有给出可直接理解的结论摘要。
6. 首次训练仍要求用户自行寻找并准备数据集，模型与数据两道门槛叠加在核心价值之前。

因此 v1.1 的产品目标不是继续增加离线工具，而是闭合以下主链路：

```text
准备模型 → 选内置示例或自己的数据 → 确认最终训练文本 → 训练 → 解释结果 → 同口径 A/B 验证 → 导出
```

## 二、目标

### 2.1 产品目标

- 用户从训练完成到看到基线 / State 对比结果，只需一次点击。
- 验证会话自动继承正确的模板、模型与 State，不要求用户记忆训练参数。
- reasoning 会话的格式参数对用户可见、可调，并由 Python 统一渲染，避免 Swift 与后端口径分裂。
- 训练前展示真实渲染文本、token 长度和截断风险。
- 训练后给出“发生了什么”的结果摘要，而不只提供曲线。
- 内置经审核且可追溯的 NekoQA 200 条示例，使首次训练只剩准备模型这一项外部前置条件。

### 2.2 成功指标

- 完成页点击“比较效果”后，首个 A/B 结果无需再次配置即可生成。
- 使用 `instruction` 训练的记录进入验证后，实际会话模板仍为 `instruction`。
- 用户选定 reasoning / think 后，App 实际下发配置的首轮 prompt token 与 Python `render_prompt` 逐 token 一致。
- 任何会清空已有会话的跨面板操作都必须经过确认。
- 有 BF16 模型时，新用户无需另备数据或手动选择输出路径即可开始训练。
- 训练前能在一个页面读到 schema、有效数、token P95 和截断数。

## 四、依赖关系

```text
P0-01 会话配置
  └─ P0-02 配置继承与元数据
       └─ P0-03 A/B 对比

P0-04 安全确认 ─────────┘

P0-05 内置示例 ─┬→ P0-03 A/B 对比
                ├→ P0-06 自动输出路径
                └→ P0-07 数据预检 → P0-08 结果摘要
P1-08 记录管理可并行实施
```

P0-03 不应在 P0-01、P0-02 未完成前独立发布，否则双栏虽然可见，但两边可能仍使用错误模板，反而放大误导。P0-05 与会话配置可以并行开发，但端到端验收必须覆盖“内置示例训练 → A/B 对比”。

## 五、不在本版本范围

- 模型广场、Hugging Face Hub 浏览器或模型下载器。
- LoRA、Bone、多卡或新的训练算法。
- 数据集编辑器和标注工具。
- 多模型同时常驻。
- 多轮 A/B 的双 cache 并行；v1.1 先保证单轮比较可靠。
- 主题、品牌配色和装饰性视觉重做。
- 自动给训练结果打“成功 / 失败”分数；健康区间尚未标定，不制造伪指标。
- 模型在线下载；内置示例只消除数据准备门槛，不代替用户准备可训练 BF16 模型。
- Developer ID 签名、Apple 公证与无终端安装；已移入 [Backlog](Backlog-签名公证与无终端安装.md)，v1.1 继续沿用当前分发与安装说明。

## 六、发布门槛

P0 八项全部通过验收后才允许发布 v1.1。签名与公证不影响本版发布，但发布说明必须继续准确描述当前安装方式。
