# P2-11：macOS 交互一致性与键盘可达

> 优先级：P2  
> 状态：已评审，待实施  
> 建议版本：v1.1.x

## 一、问题

当前核心页面已基本符合原生 macOS 习惯，但仍存在跨页面不一致：部分主操作因 `.controlSize(.large)` 呈胶囊，部分为标准圆角矩形；内容层命令较少进入菜单栏；部分面板操作缺少快捷键。

这些问题不阻断主线，但会降低产品完成度和键盘工作流效率。

## 二、目标

- 主操作视觉形态在全 App 一致。
- 常用命令可以用键盘完成。
- 打开面板、破坏性操作、默认动作遵循 macOS 语义。
- 不用自定义 ButtonStyle 破坏 Liquid Glass。

## 三、按钮规则

- 主操作：`.borderedProminent`，使用标准高度，不添加 `.controlSize(.large)`。
- 次操作：`.bordered` 或 plain，按层级而非页面自由决定。
- 破坏性操作：使用 destructive role，不用红色自定义背景。
- 打开窗口或文件面板的按钮以省略号结尾。
- 不手写 RoundedRectangle 主按钮样式。

统一检查范围：

- 对话“连接”。
- 训练“继续配置 / 开始训练”。
- 工具箱转换、量化、检查、保存。
- 失败 / 取消终态动作。

## 四、菜单与快捷键

建议加入：

- `⌘Return`：开始训练 / 发送。
- `Esc`：仅在生成或工具任务正在运行、且当前没有需要优先关闭的 sheet/menu/IME 候选界面时，停止生成或请求取消任务。
- `⌘D`：切换 A/B。
- `⌥⌘I`：显示参数或会话检查器。
- 菜单栏提供模型选择、环境诊断、欢迎窗口和当前页面主要命令。

快捷键不得抢占 TextEditor 的标准多行编辑行为。
“清空当前对话”在菜单中可达且继续确认，v1.1.x 不预设 `⌘K` 等可能与搜索/命令面板惯例冲突的快捷键。

## 五、可访问性

- 图标按钮至少有明确 accessibility label 和 help。
- 状态不只靠颜色表达。
- 重要说明不能只放 tooltip。
- 图表继续提供分段语义摘要。
- 键盘焦点顺序与视觉顺序一致。

## 六、验收标准

- [ ] 全 App 主操作使用相同标准高度形态。
- [ ] 不新增自定义主按钮 ButtonStyle。
- [ ] 所有打开面板的命令带省略号。
- [ ] A/B、参数、发送、停止可以通过键盘完成。
- [ ] VoiceOver 能读出 State、模型、会话模式和破坏性操作后果。
- [ ] 深色与浅色外观下无硬编码背景导致的对比问题。

## 七、不做

- 不引入品牌主题系统。
- 不自定义窗口 chrome。
- 不为视觉统一牺牲原生控件行为。
