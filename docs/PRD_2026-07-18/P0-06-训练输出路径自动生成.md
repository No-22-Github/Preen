# P0-06：训练输出路径自动生成

> 优先级：P0  
> 状态：已评审，待实施  
> 所属版本：v1.1.0

## 一、问题

用户选择模型和训练数据后，仍必须手动决定 `.npz` 保存位置，否则无法开始训练。输出路径是文件管理细节，不应成为主线上的必答题。

## 二、目标

- 选择数据后自动得到安全、可预测的输出路径。
- 高级用户仍可修改。
- 不覆盖已有文件。
- `.npz`、metadata、可选 `.pth` 和 run 记录保持关联。

## 三、默认路径

默认保存到：

```text
~/Library/Application Support/Preen/states/<数据名>-<模型名>-<时间>/state.npz
```

同时使用同目录：

```text
state.meta.json
state.pth       # 开启导出时
```

时间使用本地可读且文件系统安全的格式，例如 `20260718-153012`。

该目录是 Preen 管理的 State 库，不是临时缓存。删除 App 本体不会删除其中数据；“删除训练记录”仍只删 run 记录，不级联删除 State / metadata / PTH。

## 四、交互

- 配置页默认显示自动路径，并标记“自动”。
- 提供“更改…”按钮选择其他位置。
- 用户手动选择后，不因 ctx、epoch 等参数变化重置路径。
- 用户更换数据或模型时，如果路径仍处于自动模式，则重新生成；手动模式不覆盖。
- 开始训练前检查父目录可写、目标不存在、剩余磁盘空间足以完成产物的临时写入和原子替换。阻断阈值以根据 State/PTH 预估产物大小计算的需求为准，不使用没有依据的固定 GB 值，也不把可能的系统 swap 当作产物空间硬阻断。

## 五、冲突处理

- 自动路径冲突：追加短序号或新的时间戳。
- 手动路径已存在：禁止静默覆盖，提供“选择其他位置”。
- run 已创建但训练未启动成功：保留失败记录，未写出的 State 不创建空文件。

## 六、验收标准

- [ ] 有模型和数据时，无需打开保存面板即可开始训练。
- [ ] 自动路径不覆盖任何已有文件。
- [ ] 更换数据或模型后，自动路径同步更新。
- [ ] 手动路径不会被后续参数调整改写。
- [ ] 完成后的 `.npz`、metadata 和 `.pth` 在历史记录中正确关联。
- [ ] 磁盘不可写或空间不足时在启动前阻断并给出下一步。
- [ ] 删除 run 记录不会删除已产生的 State / metadata / PTH。

## 七、不做

- 不在本版本提供自定义文件名模板系统。
- 不自动移动用户手动指定的旧产物。
